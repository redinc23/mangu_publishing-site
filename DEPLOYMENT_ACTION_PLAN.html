2AWS Setup 30-45 minutes

A. Create ACM Certificate (20 min)

  1. Go to AWS Certificate Manager
  2. Click "Request certificate" ‚Üí "Request a public certificate"
  3. Enter your domain (e.g., mangu-publishing.com)
  4. Choose "DNS validation"
  5. Add the DNS records it gives you to your domain provider
  6. Wait for status to show "Issued" (5-30 min)
  7. Copy the ARN (looks like: arn:aws:acm:us-east-1:123456789:certificate/abc-123)

B. Configure Terraform (5 min)

cd infrastructure/terraform
cp terraform.tfvars.example terraform.tfvars
nano terraform.tfvars  # or use any editor

Add your values:

aws_region      = "us-east-1"
environment     = "production"
project_name    = "mangu-publishing"
domain_name     = "your-domain.com"
certificate_arn = "arn:aws:acm:us-east-1:..." # from step A
db_name         = "mangu"
db_username     = "mangu_admin"

C. Deploy Infrastructure (20 min)

terraform init
terraform plan  # Review what will be created
terraform apply  # Type 'yes' when prompted

This creates:

  • VPC with subnets
  • RDS PostgreSQL (Multi-AZ)
  • Redis ElastiCache
  • ECS Fargate cluster
  • Application Load Balancer
  • S3 buckets
  • CloudFront CDN
  • CloudWatch monitoring

D. Populate Secrets (5 min)

# Generate JWT secret
aws secretsmanager put-secret-value \
  --secret-id mangu-publishing-jwt-secret-production \
  --secret-string "$(openssl rand -base64 64)"

# Add Stripe keys (if you have them)

  --secret-id mangu-publishing-stripe-keys-production \

Common Issues & Solutions:

1. Terraform fails?

Usually it's one of these:

  • Missing ACM certificate ARN ‚Üí Create certificate first
  • AWS credentials not configured ‚Üí Run aws configure
  • Region mismatch ‚Üí Check terraform.tfvars

2. Deployment fails?

ECS Circuit Breaker will auto-rollback. Check:

  • CloudWatch logs: /ecs/mangu-publishing-server-production
  • GitHub Actions output
  • ECS service events in AWS Console
